# Changelog

Use this file to record implemented changes.

Instructions:
- Add each new change at the top of the entries list, directly below this section.
- Use this format for every entry: `DATE - TITLE - SHORT DESCRIPTION`
- Keep entries ordered from newest at the top to oldest at the bottom.
- Keep descriptions brief and focused on what was changed.
- Use an exact date in `YYYY-MM-DD` format.

Example:
- `2026-03-08 - Add changelog - Created docs/CHANGELOG.md with usage instructions and entry format guidance.`
